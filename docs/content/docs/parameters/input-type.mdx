# input_type Parameter

The `input_type` parameter optimizes embeddings for either search queries or documents/passages.

## Overview

Many providers optimize their models differently for:
- **Queries**: Short search questions or prompts
- **Documents**: Longer passages or content to be searched

Using the correct `input_type` can improve retrieval quality.

## Values

- `"query"` - For search queries, questions, prompts
- `"document"` - For passages, documents, content to be indexed

## Provider Support

| Provider | Supports input_type | Mapping |
|----------|-------------------|---------|
| Voyage AI | ✅ | Direct |
| Cohere | ✅ | → search_query / search_document |
| Gemini | ✅ | Direct |
| Jina AI | ✅ | Direct |
| Mistral AI | ✅ | Direct |
| Nomic | ✅ | → search_query / search_document |
| Mixed Bread | ✅ | Prompt injection |
| **OpenAI** | ❌ | **Parameter ignored** |
| **Cloudflare** | ❌ | **Not supported** |
| **DeepInfra** | ❌ | **Not supported** |
| **Together AI** | ❌ | **Not supported** |

## Examples

### Query Embeddings

Use `input_type="query"` for search queries:

```python
# Search query
query_embedding = client.embed(
    model="voyage-3",
    input="What is the capital of France?",
    input_type="query"
)

# Question
question_embedding = client.embed(
    model="voyage-3",
    input="How does photosynthesis work?",
    input_type="query"
)
```

### Document Embeddings

Use `input_type="document"` for passages to be searched:

```python
# Document/passage
doc_embedding = client.embed(
    model="voyage-3",
    input="Paris is the capital and largest city of France...",
    input_type="document"
)

# Content to index
content_embedding = client.embed(
    model="voyage-3",
    input="Photosynthesis is the process by which plants...",
    input_type="document"
)
```

### Asymmetric Search

For semantic search, use different `input_type` for queries and documents:

```python
# Index documents
documents = [
    "The Eiffel Tower is in Paris",
    "London is the capital of the UK",
    "Berlin is Germany's capital"
]

doc_embeddings = []
for doc in documents:
    response = client.embed(
        model="voyage-3",
        input=doc,
        input_type="document"
    )
    doc_embeddings.append(response.embeddings[0])

# Search with query
query_response = client.embed(
    model="voyage-3",
    input="What is the capital of France?",
    input_type="query"
)

# Compare query embedding with document embeddings
# (use cosine similarity)
```

## Provider-Specific Behavior

### Cohere & Nomic

These providers map `input_type` to their internal parameters:

```python
# input_type="query" → search_query
# input_type="document" → search_document

response = client.embed(
    model="embed-v4.0",  # Cohere
    input="Query",
    input_type="query"  # Becomes search_query
)
```

### OpenAI

OpenAI **ignores** `input_type`:

```python
# This works but input_type is ignored
response = client.embed(
    model="text-embedding-3-small",
    input="Text",
    input_type="query"  # ⚠️ Ignored by OpenAI
)
```

### Mixed Bread

Mixed Bread uses `input_type` for prompt injection to optimize the model.

## When to Use

**Use `input_type="query"`** for:
- Search queries
- Questions
- Short prompts
- User inputs

**Use `input_type="document"`** for:
- Documents to be indexed
- Long-form content
- Knowledge base articles
- Passages

**Skip `input_type`** for:
- Symmetric similarity tasks
- Clustering
- Classification
- Providers that don't support it

## Best Practices

- Always use `input_type` when supported by your provider
- Use "query" for search/question inputs
- Use "document" for content being searched
- Check provider support before relying on this parameter
- For symmetric tasks (similarity, clustering), either value works
