# dimensions Parameter

The `dimensions` parameter allows you to control the size of embedding vectors using Matryoshka Representation Learning.

## What are Matryoshka Embeddings?

Matryoshka embeddings allow you to use smaller-dimensional embeddings from models trained to preserve information across multiple scales. This enables:

- **Faster similarity search** - Fewer dimensions to compare
- **Lower storage costs** - Smaller vectors to store
- **Flexible trade-offs** - Balance quality vs efficiency

## Examples

### Voyage AI

```python
# Default dimensions (1024)
full_response = client.embed(
    model="voyage-3",
    input="Sample text"
)
print(full_response.dimensions)  # 1024

# Reduced dimensions
small_response = client.embed(
    model="voyage-3",
    input="Sample text",
    dimensions=256
)
print(small_response.dimensions)  # 256

# Supported values: 256, 512, 1024, 2048
```

### Gemini

```python
# Wide range of dimensions (128-3072)
response = client.embed(
    model="gemini-embedding-001",
    input="Sample text",
    dimensions=512
)
```

### OpenAI (text-embedding-3 only)

## Benefits of Lower Dimensions

### Storage Savings

```python
# 1024 dimensions vs 256 dimensions
full_embedding = [0.1] * 1024   # 4KB per vector (float32)
small_embedding = [0.1] * 256   # 1KB per vector (float32)

# 75% storage reduction!
```

### Faster Similarity Search

```python
import time
import numpy as np

# Simulate similarity search
def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# 1024-d vectors
large_vecs = [np.random.rand(1024) for _ in range(10000)]
start = time.time()
for vec in large_vecs[:100]:
    similarities = [cosine_similarity(vec, v) for v in large_vecs]
elapsed_large = time.time() - start

# 256-d vectors
small_vecs = [np.random.rand(256) for _ in range(10000)]
start = time.time()
for vec in small_vecs[:100]:
    similarities = [cosine_similarity(vec, v) for v in small_vecs]
elapsed_small = time.time() - start

print(f"1024-d: {elapsed_large:.2f}s")
print(f"256-d: {elapsed_small:.2f}s")  # Much faster!
```

## Trade-offs

### Smaller Dimensions
- ✅ Faster similarity search
- ✅ Lower storage costs
- ✅ Reduced memory usage
- ⚠️ Slightly lower retrieval quality

### Full Dimensions
- ✅ Best retrieval quality
- ✅ Maximum information preservation
- ⚠️ Higher storage costs
- ⚠️ Slower search

## Choosing Dimensions

**Use smaller dimensions (256-512) when:**
- Storage cost is a concern
- Search speed is critical
- Working with millions of vectors
- Slight quality reduction is acceptable

**Use full dimensions (1024+) when:**
- Maximum quality is required
- Dataset is small (under 100K vectors)
- Storage cost isn't a concern

**Test both** to find the sweet spot for your use case.
